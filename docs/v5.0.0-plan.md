# OpenClaw PM v5.0.0 开发计划

## 🎯 核心目标

**聚焦**：性能优化 + 代码质量 + 开发体验
**不做**：Web Dashboard、实时监控、告警系统

## 📋 功能清单

### 1. 性能优化（高优先级）

#### 1.1 日志文件索引
**问题**：每次统计都要全文扫描日志文件（慢）
**方案**：
```typescript
// 创建日志索引
interface LogIndex {
  date: string;
  lineCount: number;
  messageCount: number;
  errorCount: number;
  hourlyDistribution: { hour: number; offset: number; count: number }[];
  lastModified: number;
}

// 索引文件：~/.openclaw/pm-cache/log-index-2026-04-15.json
// 首次扫描：全文扫描 + 生成索引
// 后续查询：读取索引（快 10-100 倍）
```

**收益**：
- 统计速度提升 10-100 倍
- 减少磁盘 I/O
- 支持大日志文件（>100MB）

#### 1.2 增量分析
**问题**：每次都重新分析整个日志文件
**方案**：
```typescript
// 记录上次分析位置
interface AnalysisCheckpoint {
  logFile: string;
  lastOffset: number;
  lastTimestamp: string;
  stats: Partial<DailyStats>;
}

// 只分析新增内容
// 合并增量结果
```

**收益**：
- 实时统计成本降低 90%
- 支持持续监控
- 减少 CPU 占用

#### 1.3 缓存机制
**问题**：重复查询相同数据
**方案**：
```typescript
// 内存缓存 + 磁盘缓存
class CacheManager {
  private memoryCache: Map<string, any>;
  private diskCachePath: string;
  
  get(key: string, ttl: number): any;
  set(key: string, value: any): void;
  invalidate(pattern: string): void;
}

// 缓存策略
- 每日统计：缓存 24 小时
- 晨间简报：缓存 1 小时
- 心跳检查：不缓存
```

**收益**：
- 重复查询速度提升 100 倍
- 减少磁盘读取
- 降低系统负载

#### 1.4 并发处理
**问题**：串行处理多个检查项（慢）
**方案**：
```typescript
// 并发执行检查项
async check(): Promise<HealthCheckResult> {
  const [gateway, sessions, queue, providers, cron] = await Promise.all([
    this.checkGateway(),
    this.checkSessions(),
    this.checkQueue(),
    this.checkProviders(),
    this.checkCron(),
  ]);
  
  return { gateway, sessions, queue, providers, cron };
}
```

**收益**：
- 健康检查速度提升 3-5 倍
- 更好的用户体验
- 充分利用多核 CPU

### 3. 已知限制修复（中优先级）

#### 3.1 日志文件检测
**问题**：日志不存在时报错不友好
**方案**：
```typescript
private getLogFile(date: string): string {
  const logPath = path.join(this.config.get('openclaw.dir'), 'logs', `gateway-${date}.log`);
  
  if (!fs.existsSync(logPath)) {
    // 尝试其他可能的日志路径
    const altPaths = [
      path.join(this.config.get('openclaw.dir'), 'agents/main/logs', `${date}.log`),
      path.join(this.config.get('openclaw.dir'), `gateway-${date}.log`),
    ];
    
    for (const altPath of altPaths) {
      if (fs.existsSync(altPath)) {
        this.logger.info(`使用备用日志路径: ${altPath}`);
        return altPath;
      }
    }
    
    throw new Error(
      `日志文件不存在: ${logPath}\n` +
      `提示: 请检查 OpenClaw Gateway 是否正在运行，或指定正确的日志路径`
    );
  }
  
  return logPath;
}
```

#### 3.2 配置文件自动生成
**问题**：首次使用需要手动创建配置
**方案**：
```typescript
class ConfigManager {
  constructor(configPath?: string) {
    this.configPath = configPath || this.getDefaultConfigPath();
    
    if (!fs.existsSync(this.configPath)) {
      this.logger.info('配置文件不存在，创建默认配置...');
      this.createDefaultConfig();
    }
    
    this.load();
  }
  
  private createDefaultConfig(): void {
    const defaultConfig = {
      openclaw: {
        dir: path.join(os.homedir(), '.openclaw'),
        sessions_dir: path.join(os.homedir(), '.openclaw/agents/main/sessions'),
        gateway_port: 3000,
      },
      health_check: {
        max_queue_age_hours: 2,
        provider_error_threshold: 10,
      },
      cron_tasks: [],
    };
    
    fs.mkdirSync(path.dirname(this.configPath), { recursive: true });
    fs.writeFileSync(this.configPath, JSON.stringify(defaultConfig, null, 2));
    this.logger.success(`已创建默认配置: ${this.configPath}`);
  }
}
```

#### 3.3 错误处理增强
**问题**：错误信息不够详细
**方案**：
```typescript
// 自定义错误类
class OpenClawPMError extends Error {
  constructor(
    message: string,
    public code: string,
    public details?: any
  ) {
    super(message);
    this.name = 'OpenClawPMError';
  }
}

// 错误处理中间件
function handleError(error: Error): void {
  if (error instanceof OpenClawPMError) {
    logger.error(`[${error.code}] ${error.message}`);
    if (error.details) {
      logger.debug('详细信息:', error.details);
    }
  } else {
    logger.error('未知错误:', error);
  }
  
  // 提供解决建议
  const suggestions = getErrorSuggestions(error);
  if (suggestions.length > 0) {
    logger.info('建议:');
    suggestions.forEach(s => logger.info(`  - ${s}`));
  }
}
```
