### 1. 新增功能：健康检查历史追踪

#### health-history.sh
**文件**: `scripts/health-history.sh` (9951 字节)

**功能**:
- **记录健康检查结果**: 自动记录每次健康检查的结果
- **查询历史记录**: 支持按天数过滤（默认 7 天）
- **统计分析**: 计算健康评分、平均错误数、平均 Lock 数等
- **自动清理**: 清理 30 天前的旧记录
- **JSON 输出**: 支持 JSON 格式输出，便于集成

**数据结构**:
```json
{
  "timestamp": "2026-04-15T05:30:00Z",
  "date": "2026-04-15",
  "time": "13:30:00",
  "gateway": {
    "running": true,
    "count": 1
  },
  "locks": {
    "count": 0
  },
  "queue": {
    "stuck": false
  },
  "provider": {
    "errors": 0
  }
}
```

**使用方法**:
```bash
# 查看最近 7 天的历史
./scripts/health-history.sh list

# 查看最近 30 天的历史
./scripts/health-history.sh list --days 30

# 显示统计信息
./scripts/health-history.sh stats

# 清理 30 天前的记录
./scripts/health-history.sh clean

# JSON 格式输出
./scripts/health-history.sh stats --json
```

**统计指标**:
- 总检查次数
- Gateway 停止次数
- 多个 Gateway 次数
- 总 Lock 数
- 总错误数
- 平均 Lock/次
- 平均错误/次
- **健康评分** (0-100)

**健康评分算法**:
```
初始分数: 100
- Gateway 停止: -10 分/次
- 多个 Gateway: -5 分/次
- Lock 数 > 10: -10 分
- 错误数 > 50: -20 分
最低分数: 0
```

**集成**:
- 自动集成到 `gateway-health-check.sh`
- 每次健康检查后自动记录结果
- 数据存储在 `~/.openclaw/health-history/health-checks.jsonl`

### 2. 修复：thinking-only session 清理逻辑

#### 问题描述
**原问题**: 当检测到 thinking-only session（只有 thinking，没有 text）时，直接删除最后一条消息，导致数据丢失。

**优先级**: 中

#### 解决方案
**改为标记而非删除**:
1. 在消息中添加 `_incomplete` 字段（标记为不完整）
2. 添加 `_marked_at` 字段（记录标记时间）
3. 保留原始消息数据
4. 如果 jq 不可用，回退到删除方式

**修改文件**: `scripts/gateway-health-check.sh`

**修改前**:
```bash
# 备份后删除最后一条消息
create_backup "$session_file" "thinking-only" > /dev/null
head -n -1 "$session_file" > "$session_file.tmp"
mv "$session_file.tmp" "$session_file"
```

**修改后**:
```bash
# 备份原文件
create_backup "$session_file" "thinking-only" > /dev/null

# 标记为 incomplete 而非删除
local marked_msg=$(echo "$last_msg" | jq '. + {"_incomplete": true, "_marked_at": "'$(date -u +"%Y-%m-%dT%H:%M:%SZ")'"}' 2>/dev/null)

if [ -n "$marked_msg" ]; then
    # 替换最后一行
    head -n -1 "$session_file" > "$session_file.tmp"
    echo "$marked_msg" >> "$session_file.tmp"
    mv "$session_file.tmp" "$session_file"
    log "Marked thinking-only message as incomplete (not deleted)"
else
    # jq 失败，回退到删除方式
    log "Warning: jq not available, removing message instead"
    head -n -1 "$session_file" > "$session_file.tmp"
    mv "$session_file.tmp" "$session_file"
fi
```

**效果**:
- ✅ 保留原始消息数据
- ✅ 可以追溯标记时间
- ✅ 避免数据丢失
- ✅ 兼容 jq 不可用的情况

### 3. 修复：自动恢复功能改进

#### 问题描述
**原问题**: `check-unanswered.sh --recover` 只发送 wake 通知，无法直接向 session 发送消息。

**优先级**: 低

#### 解决方案
**优先使用 sessions_send API**:
1. 尝试使用 `openclaw sessions send` 命令
2. 如果不可用，回退到 wake 通知
3. 显示详细的恢复状态（成功/失败）
4. 改进错误处理和用户反馈

**修改文件**: `scripts/check-unanswered.sh`

**修改前**:
```bash
# 发送恢复消息（通过 sessions_send）
# 注意：这需要 OpenClaw CLI 支持
if ! $JSON_OUTPUT; then
    echo -e "  ${CYAN}→${NC} $session_key"
fi

# 这里应该调用 OpenClaw API 发送消息
# 由于没有直接的 API，我们发送 wake 通知让 agent 处理
send_wake_notification "[未回复消息恢复] 检测到 session $session_key 有未回复的消息，请检查并回复" "now"
```

**修改后**:
```bash
local recovered=0
local failed=0

for session_key in "${UNANSWERED_SESSIONS[@]}"; do
    # 提取 agent 和 session
    local agent=$(echo "$session_key" | cut -d: -f2)
    local session=$(echo "$session_key" | cut -d: -f3)
    
    if ! $JSON_OUTPUT; then
        echo -e "  ${CYAN}→${NC} $session_key"
    fi
    
    # 尝试使用 OpenClaw CLI 发送消息
    if check_command "openclaw"; then
        # 构造恢复消息
        local recovery_msg="[自动恢复] 检测到有未回复的消息，正在处理..."
        
        # 尝试通过 OpenClaw CLI 发送（如果支持）
        if openclaw sessions send --session "$session_key" --message "$recovery_msg" 2>/dev/null; then
            ((recovered++))
            if ! $JSON_OUTPUT; then
                echo -e "    ${GREEN}✓${NC} 已发送恢复消息"
            fi
        else
            # 回退到 wake 通知
            send_wake_notification "[未回复消息恢复] session $session_key 有未回复的消息" "now"
            ((recovered++))
            if ! $JSON_OUTPUT; then
                echo -e "    ${YELLOW}⚠${NC} 已发送 wake 通知（sessions_send 不可用）"
            fi
        fi
    else
        # OpenClaw CLI 不可用，使用 wake 通知
        send_wake_notification "[未回复消息恢复] session $session_key 有未回复的消息" "now"
        ((recovered++))
        if ! $JSON_OUTPUT; then
            echo -e "    ${YELLOW}⚠${NC} 已发送 wake 通知（openclaw CLI 不可用）"
        fi
    fi
done

if ! $JSON_OUTPUT; then
    if [ $recovered -gt 0 ]; then
        echo -e "${GREEN}✓ 已处理 $recovered 个 session${NC}"
    fi
    if [ $failed -gt 0 ]; then
        echo -e "${RED}✗ $failed 个 session 处理失败${NC}"
    fi
fi
```

**效果**:
- ✅ 优先使用 `openclaw sessions send` 命令
- ✅ 如果不可用，回退到 wake 通知
- ✅ 显示详细的恢复状态
- ✅ 改进错误处理和用户反馈
