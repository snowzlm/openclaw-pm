# OpenClaw PM v4.0.0 迁移指南

## 概述

OpenClaw PM v4.0.0 是一次重大重构，从 Bash 脚本迁移到 TypeScript。本指南帮助你从 v3.x 平滑迁移到 v4.0.0。

---

## 快速开始

### 1. 安装 v4.0.0

```bash
cd /path/to/openclaw-pm
git pull origin main
npm install
npm run build
npm link
```

### 2. 验证安装

```bash
openclaw-pm --version
# 输出: 4.0.0

openclaw-pm --help
# 显示命令列表
```

### 3. 运行健康检查

```bash
openclaw-pm health
```

## 主要变化

### 1. 命令行接口

#### v3.x (Bash)
```bash
./scripts/gateway-health-check.sh
./scripts/quick-diagnose.sh
./scripts/morning-briefing.sh
```

#### v4.0.0 (TypeScript CLI)
```bash
openclaw-pm health
openclaw-pm diagnose  # 未来版本
openclaw-pm briefing  # 未来版本
```

### 2. 配置管理

#### v3.x
```bash
# 配置文件: scripts/config.json
{
  "openclaw_dir": "~/.openclaw",
  "gateway_port": 3000
}
```

#### v4.0.0
```bash
# 配置文件: ~/.openclaw/pm-config.json
{
  "openclaw": {
    "dir": "~/.openclaw",
    "gateway_port": 3000
  }
}
```

### 3. 输出格式

#### v3.x
```
=== Gateway Health Check ===
Status: OK
Issues: 0
```

#### v4.0.0
```
=== OpenClaw Gateway 健康检查 ===

状态: HEALTHY
评分: 95/100
时间: 2026-04-15 13:59:00

检查项:
  ✓ gateway: Gateway 运行正常
  ✓ sessions: Sessions 正常 (12 个)
```

## 迁移步骤

### 步骤 1: 备份现有配置

```bash
# 备份 v3.x 配置
cp scripts/config.json scripts/config.json.backup

# 备份 Cron 任务
crontab -l > crontab.backup
```

### 步骤 2: 安装 v4.0.0

```bash
# 拉取最新代码
git pull origin main

# 安装依赖
npm install

# 构建 TypeScript
npm run build

# 全局链接
npm link
```

### 步骤 3: 创建新配置

```bash
# 方式 1: 使用默认配置
openclaw-pm config > ~/.openclaw/pm-config.json

# 方式 2: 手动创建
cat > ~/.openclaw/pm-config.json << 'EOF'
{
  "openclaw": {
    "dir": "~/.openclaw",
    "gateway_port": 3000,
    "gateway_timeout": 30
  },
  "health_check": {
    "interval_minutes": 5,
    "max_lock_age_hours": 1,
    "max_queue_age_hours": 2,
    "provider_error_threshold": 10
  },
  "backup": {
    "enabled": true,
    "max_backups": 10,
    "backup_dir": "~/.openclaw/.backup"
  }
}
EOF
```

### 步骤 4: 测试新命令

```bash
# 测试健康检查
openclaw-pm health

# 测试 JSON 输出
openclaw-pm health --json

# 测试详细输出
openclaw-pm health --verbose

# 测试备份
openclaw-pm backup
```

### 步骤 5: 更新 Cron 任务

```bash
# 编辑 Cron
crontab -e

# 旧方式（v3.x）
# */5 * * * * /path/to/openclaw-pm/scripts/gateway-health-check.sh

# 新方式（v4.0.0）
*/5 * * * * openclaw-pm health >> ~/.openclaw/logs/health-check.log 2>&1
```

### 步骤 6: 验证迁移

```bash
# 等待 5 分钟，检查日志
tail -f ~/.openclaw/logs/health-check.log

# 检查 Cron 是否执行
grep "openclaw-pm" /var/log/syslog
```

## 常见问题

### Q1: v3.x 脚本还能用吗？

**A**: 可以。v3.x 脚本保留在 `scripts/` 目录，可通过 npm scripts 调用：

```bash
npm run legacy:health
npm run legacy:diagnose
npm run legacy:briefing
```

### Q2: 如何回退到 v3.x？

**A**: 使用 Git 回退：

```bash
git checkout v3.1.0
npm install
```

### Q3: v4.0.0 支持哪些平台？

**A**: 
- ✅ Linux (Ubuntu, Debian, CentOS)
- ✅ macOS (Intel, Apple Silicon)
- ❌ Windows (未测试)

### Q4: 配置文件在哪里？

**A**: 
- 默认: `~/.openclaw/pm-config.json`
- 自定义: `openclaw-pm --config /path/to/config.json`

### Q5: 如何查看详细日志？

**A**: 使用 `--verbose` 或 `--debug` 选项：

```bash
openclaw-pm health --verbose
openclaw-pm health --debug
```

### Q6: 如何集成到现有监控系统？

**A**: 使用 JSON 输出：

```bash
openclaw-pm health --json | jq '.status'
```

### Q7: 如何自定义健康检查？

**A**: 编辑配置文件：

```json
{
  "health_check": {
    "interval_minutes": 5,
    "max_lock_age_hours": 1,
    "max_queue_age_hours": 2,
    "provider_error_threshold": 10
  }
}
```

## 故障排查

### 问题 1: `openclaw-pm: command not found`

**原因**: 未全局链接

**解决**:
```bash
cd /path/to/openclaw-pm
npm link
```

### 问题 2: `Cannot find module 'commander'`

**原因**: 依赖未安装

**解决**:
```bash
npm install
```

### 问题 3: TypeScript 编译错误

**原因**: TypeScript 版本不兼容

**解决**:
```bash
npm install typescript@latest --save-dev
npm run build
```

### 问题 4: 健康检查失败

**原因**: Gateway 未运行或配置错误

**解决**:
```bash
# 检查 Gateway 状态
openclaw gateway status

# 检查配置
openclaw-pm config

# 使用调试模式
openclaw-pm health --debug
```

## 最佳实践

### 1. 使用配置文件

```bash
# 创建项目专用配置
cat > ~/.openclaw/pm-config.json << 'EOF'
{
  "openclaw": {
    "dir": "~/.openclaw",
    "gateway_port": 3000
  }
}
EOF
```

### 2. 定期备份

```bash
# 每天凌晨 3 点自动备份
0 3 * * * openclaw-pm backup --type auto
```

### 3. 监控健康状态

```bash
# 每 5 分钟检查一次
*/5 * * * * openclaw-pm health --json >> ~/.openclaw/logs/health.jsonl
```

### 4. 集成到 CI/CD

```yaml
# .github/workflows/health-check.yml
name: Health Check
on:
  schedule:
    - cron: '*/5 * * * *'
jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - run: npm install
      - run: npm run build
      - run: openclaw-pm health --json
```
